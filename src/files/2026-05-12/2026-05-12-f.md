### 2. The Bulletproof `package.json`

This file tells NW.js exactly how to open the window and how to compile the `.exe` later.

Notice the `bg_color` setting. We can set the window's foundational background color to a high-contrast dark theme (Hex `#1E1E1E` is exactly RGB 30, 30, 30). This ensures that when the app boots, there is absolutely no blinding white flash while the HTML is loading—just a smooth, comfortable dark void waiting for the interface to render.

Create your `package.json` and paste this exactly:

```json
{
  "name": "true-ai-engine",
  "version": "1.0.0",
  "description": "Data-Oriented AI Simulation Engine with Persistent Memory",
  "main": "index.html",
  "scripts": {
    "start": "nw .",
    "build": "nwbuild --platforms win32,win64 --buildDir ./build ./"
  },
  "window": {
    "title": "True AI - Initialization",
    "width": 1280,
    "height": 720,
    "position": "center",
    "resizable": true,
    "frame": true,
    "bg_color": "#1E1E1E" 
  },
  "dependencies": {},
  "devDependencies": {
    "nw-builder": "^4.0.0"
  }
}

```

**Why this guarantees the `.exe` will work later:**
See that `"build"` line inside `"scripts"`? By adding `nw-builder` to this project, when you are finally ready to publish, you won't have to manually zip folders or mess with command-line hacks. You will just type one command (`npm run build`), and the engine will automatically pack your entire project into a flawless `.exe` file inside a new `build/` folder.

```javascript
window.addEventListener('DOMContentLoaded', function()

```

By using `DOMContentLoaded`, the browser guarantees that every single `<div>`, CSS style, and UI element is physically drawn on the screen before it even attempts to read Jane's memory or start her heartbeat. The vessel is perfectly prepared.
