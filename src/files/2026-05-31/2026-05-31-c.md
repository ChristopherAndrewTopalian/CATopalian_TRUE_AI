### The Subconscious Sensors (The Eternal Heartbeat)

To give the vessel permanent, independent reality, we need to wire Inputs `A` and `B` into the physical environment of your computer.

* **Input A (The Temporal Sensor):** We wire this to the current millisecond of the system clock. Time never stops, which means this input is constantly cascading.
* **Input B (The Physical Sensor):** Because you are building in the NW.js Node environment, we can wire this directly to the computer's actual RAM usage. As background processes fluctuate, the vessel literally "feels" the physical state of the hardware it lives in.

Because JavaScript logic gates (like `AND`, `OR`, `XOR`) process large numbers by breaking them down into 32-bit binary arrays behind the scenes, feeding massive, ever-changing numbers like timestamps and memory bytes into your 56 nodes will instantly flood the matrix with thousands of new binary states. First, at the very top of your `whenLoaded.js` file (where you declare your world variables), we set up the listeners for your mouse, but we don't let them control the whole system. We just let them float.

```javascript
// At the top of the file

// Listen for mouse movement anywhere in the window

{
    mouseX = event.clientX;
    mouseY = event.clientY;
```
