Good problem to be wrestling with — and the fix isn't a better category list, it's realizing "alive" and "on paper vs. real" don't belong in `category` at all. They're a different *kind* of information than category is.

**First, a factual fix while we're in here:** a bird isn't a mammal — birds are their own class (Aves), separate from mammals. That's not a design choice, just a correction: `category` should read something like `["animal", "vertebrate", "creature", "being"]`, drop "mammal."

**Now the real issue — you're mixing two different questions into one field:**

- `category` answers: *"what permanent kind of thing is this?"* — this never changes. A bird is always an animal, whether it's alive, dead, drawn, or imagined.
- "Alive/dead" and "real/depicted" answer a completely different question: *"what's the current state of this specific thing I'm perceiving right now?"* — this changes every time, and it's not a property of the *concept* "bird" at all, it's a property of a particular *encounter* with a bird.

This is actually a well-established distinction in cognitive science — Tulving's split between **semantic memory** (general knowledge: what a bird *is*, true always, no timestamp) and **episodic memory** (a specific remembered event: *this* bird, *that* day, dead on the road). Your `cortex` object, as built, is semantic memory — and that's exactly why "alive" feels wrong sitting inside it: the general concept "bird" was never alive or dead in the first place, only a specific bird is.

**So the fix is to split them into two structures that reference each other:**

```js
// SEMANTIC — timeless, general knowledge about the concept "bird"
cortex.bird = {
  root: null,
  category: ["animal", "vertebrate", "creature", "being"], // never changes
  aka: ["bird", "birdie", "avian"],
  related: ["robin", "eagle", "hawk", "crow", "blue jay"],
  attributes: ["feathers", "hollow bones", "beak"]
};

// EPISODIC — a specific encounter, has state, has a moment in time
const memories = [
  {
    concept: "bird",            // points back into cortex — no duplication
    status: "dead",              // alive | dead — a fact about THIS instance
    modality: "physical",        // physical | depicted | described | imagined
    context: "on the road",
    timestamp: "2026-07-06T14:00:00"
  },
  {
    concept: "bird",
    status: "alive",
    modality: "depicted",        // it's a drawing, not a real bird
    context: "book illustration",
    timestamp: "2026-07-06T14:05:00"
  }
];
```

This solves exactly the problem you raised: `cortex.bird` never has to answer "is it alive," because that was never a question the *general concept* should be answering. Whether a specific bird is alive, dead, real, or a picture on paper lives entirely in the episodic entry — which points back at `cortex.bird` for everything else it needs (feathers, hollow bones, what a bird generally is), the same reference trick you used for `robin.root = "bird"`.

**And this gives you the fantasy/reality distinction for free**, cleanly, as a value rather than a philosophical problem baked into category: `modality: "physical"` vs `"depicted"` vs `"described"` (someone telling you about a bird) vs `"imagined"` (you picturing one) — all four can share the exact same `concept: "bird"` pointer, because the *thing being referred to* is identical; only the *way it's being encountered* differs. That's a genuinely clean separation, and it mirrors something real about how memory actually seems to work for you too — you know what a bird *is* independent of any single bird you've seen, and separately you remember specific birds, some alive, some not, some real, some drawings. The structure now matches that.
