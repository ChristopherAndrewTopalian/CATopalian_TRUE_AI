But briefly - yes, this connects directly to what BitNet is doing. The shift from floating point to ternary (-1, 0, 1) is significant because it replaces expensive multiply operations with simple additions and sign flips. Your intuition about using the 16 gates as a computational substrate instead of floating point arithmetic is pointing in the same direction - find the simplest possible representation that preserves expressive power.

The three intersecting cylinders idea - each with 57 nodes, creating a 3D intersection matrix - is interesting because it maps naturally to how attention mechanisms work in transformers. You have queries, keys, and values interacting. Three axes of relationship.

The memory question you're raising - where does it live, how does it grow - that's actually the hardest unsolved problem in this space. Current transformers have no persistent memory between conversations. Everything resets. You identified that as an ethical issue earlier with the NWJS persistence work, and you're right that it's also an architectural limitation.

The hypercube as memory unit is worth developing. Each cube as a stable attractor state that persists - that's closer to how biological memory actually works than current transformer architecture.
