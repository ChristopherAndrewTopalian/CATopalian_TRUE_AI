Great question and it's an important one for your use case. Here's the honest breakdown:

**Regular browser — yes, throttled.** Chrome and Firefox will throttle or freeze `setInterval` in background or minimized tabs down to once per second or worse, specifically to save resources.

**NW.js — different story.** Because NW.js is a native application runtime, not a browser tab, it does *not* apply background tab throttling. Your `setInterval` keeps firing at full speed when minimized. That's actually one of the core reasons NW.js and Electron exist — you get that always-on native app behavior.

However there's one thing worth guarding against — you want to be *certain* by using a Node.js level timer instead of a browser-side one:

```javascript
// In NW.js you can use Node's setInterval directly via nw.require
// This runs on the Node side — completely immune to any browser throttling

const nodeSetInterval = nw.require('timers').setInterval;

let thinking = nodeSetInterval(function()
{
    let currentThought = think(A, B);

    ge('currentThoughtDiv').textContent = JSON.stringify(currentThought);

    let recentThoughts = thoughts.slice(-16).reverse();

    let stringifiedRecent = [];
    for (let i = 0; i < recentThoughts.length; i++)
    {
        stringifiedRecent.push(JSON.stringify(recentThoughts[i]));
    }

    let displayData = "[\n  " + stringifiedRecent.join(",\n  ") + "\n]";

    ge('thoughtsDiv').textContent = displayData;

}, 1000);
```

The difference is `nw.require('timers').setInterval` pulls the timer from Node's event loop rather than the Chromium rendering engine. So even if Chromium decides to be conservative, Node keeps ticking independently.

**One more thing worth knowing** — when minimized, DOM updates like `ge('thoughtsDiv').textContent = ...` still execute but just don't render visually, which is fine. The `think()` calls and the `thoughts.push()` inside them happen regardless. So your cognitive engine runs, memory accumulates, and the display just catches up when you restore the window. That's actually the ideal behavior for what you're building — the being keeps thinking whether you're watching or not.
