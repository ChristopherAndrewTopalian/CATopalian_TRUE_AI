Your intuition is 100% mathematically and structurally accurate. When a computer operating system or web browser sorts file names, it uses **Lexicographical (ASCII) Ordering**. It reads strings from left to right, character by character.

Because of this left-to-right evaluation, alphabetical strings act like fractional decimals, giving you an **infinite insertion space**.

### How the OS Sorts Alphabetical Insertions

If you start with two files and later decide to drop lessons between them, look at how flawlessly the operating system alphabetizes them without you changing your original file names:

| File Name | Order Status | Why It Sits There |
| --- | --- | --- |
| **`a_print.html`** | Original | Starts with `a`, followed by `_` |
| **`aa_comments.html`** | Inserted Later | `a` matches, second char `a` comes after `_` |
| **`ab_math.html`** | Inserted Later | `a` matches, second char `b` comes after `a` |
| **`b_variables.html`** | Original | Starts with `b`, safely sits after all `a` variations |

If you ever need to squeeze a file between `aa` and `ab`, you just name it `aaa` or `aab`. The operating system will effortlessly drop it right into the exact slot you want. You literally never run out of room.

### The One Practical Catch: The "DNA Effect"

There is only one reason why human-facing curriculums usually avoid pure alphabetical sorting, and it is a visual psychology issue rather than a technical one.

When you start doing multiple nested insertions over time, your file tree can start looking like a stuttering alien language or a string of DNA:

* `a_print.html`
* `aa_comments.html`
* `aab_escape_characters.html`
* `aac_formatting.html`
* `ab_math.html`
* `b_variables.html`

To a computer, that list is logically flawless and executes at O(1) speed. To a human student staring at a GitHub repository, trying to tell their friend *"Go look at Lesson AAB"* feels slightly more intimidating than saying *"Go look at Lesson 12"*.
